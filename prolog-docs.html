1. Overview & Architecture

ProLog is a clinical case logbook application designed for fertility and gynaecology specialists. It provides a secure, structured interface for recording surgical and clinical encounters, tracking outcomes, running analytics, and generating AI-assisted reflective summaries.

Tech Stack

React 18 Frontend UI Vite Build tool React Router 6 SPA routing Recharts Analytics charts FastAPI Python backend SQLAlchemy (async) ORM PostgreSQL 16 Database Docker Compose Containers Nginx Reverse proxy Claude Opus 4.6 AI analysis

High-Level Architecture

Browser (React SPA)
    │  HTTP/HTTPS (cookie auth)
    ▼
Nginx reverse proxy  ←── SSL termination (Let's Encrypt)
    │              │
    │  /           │  /api/*
    ▼              ▼
React build     FastAPI  (port 8001)
(static files)      │
                    ├── Auth router
                    ├── Cases router
                    ├── Users router
                    ├── AI router      ──→  Anthropic API (Claude)
                    └── Export router
                         │
                         ▼
                    PostgreSQL 16

Authentication

  • JWT stored in httpOnly cookies
  • bcrypt + SHA-256 password hashing
  • Rate-limited auth endpoints
  • Token-based password reset via email

User Isolation

  • Every DB query scoped to current_user.id
  • Users cannot access other users' cases
  • Hospital/qualification data per-user

Clinical Features

  • 3 procedure categories: Obs, Gynae, ART
  • 4-level procedure hierarchy
  • ART pregnancy outcome tracking
  • Complication recording

Export & Analytics

  • CSV and styled Excel download
  • Weekly / 8-week trend charts
  • Category distribution donut
  • AI reflective summary (Claude)

3.2 Auth Context — context/AuthContext.jsx

Provides user, loading, login(), and logout() to the entire tree. On mount it calls GET /api/auth/me to restore session. Listens globally for 401 events to auto-clear state.

3.3 API Client — api/client.js

All HTTP traffic goes through a centralised client. Every request uses credentials: 'include' so the browser sends the auth cookie automatically.

3.4 Procedure Data — data/procedures.js

A static 4-level hierarchy used to drive cascading dropdowns in the case form. The same structure is mirrored in backend/procedures.py for server-side validation.

CategoryTypes (sample)
Obstetrics Antenatal, Intrapartum, Postnatal
Gynaecology Hysteroscopy, Laparoscopy, Laparotomy, Cervical Surgery, Urogynaecology, Vaginal Surgery, Perineal Surgery, Contraception, Pregnancy Management, Cystoscopy, Surgical Wound Debridement (+2)
ART Oocyte Collection, Embryo Transfer, Male Reproductive Surgery

Exported helpers:

  • getProcTypes(category) — list of types for a category
  • getProcProcedures(category, type) — procedures for a type
  • getProcDetails(category, type, procedure) — detail options (or null)
  • isTerminalType(category, type) — true if no further level exists

3.5 Pages

Auth.jsx

Three-tab screen covering Login, Register (with auto-generated username), and Forgot Password.

Dashboard.jsx

Home screen with four stat cards (Total Cases, This Month, Obstetrics, ART), ART pregnancy-check reminders, a category donut chart, and a recent-cases list. Supports date-range and category filters.

Cases.jsx

Paginated, searchable case list. Supports search, category filter, pregnant-outcome filter, and per-page size (10 / 25 / 50 / 100). Cases can be selected for bulk delete. Includes a detail modal, per-row Edit/Delete, and Export to CSV/Excel buttons.

CaseForm.jsx

The primary data entry form, divided into conditional sections:

Always shown

  • Patient ID, Date, Hospital (autocomplete)
  • Category → Type → Procedure → Detail (cascade)
  • Clinical Role (Primary / Assistant / Supervisor)
  • Clinical notes

Obstetrics only

  • Caesarean: prev CS count, sterilisation checkbox
  • Complications: PPH, Shoulder Dystocia, pH, 3rd-degree tear, Other

ART only

  • Oocyte: trigger, hours, follicles, eggs retrieved, LA
  • Embryo Transfer: number transferred
  • Pregnancy toggle (sets 14-day check reminder)

Other conditional

  • Termination: gestation weeks
  • Cystoscopy: catheterisation

Analytics.jsx

Charts and statistics: weekly spread bar chart + daily grid, 8-week trend bars, obstetrics outcome stats, and category distribution.

AI.jsx

Sends all user cases to the backend for Claude analysis. Supports prompt types: summary, competency, and reflective. Displays the result in a formatted text box. Includes a data-privacy notice.

Profile.jsx

Account management: edit name / username / email / clinical role, change password, and manage a qualifications list.

ResetPassword.jsx

Token-based password reset page. Reads the reset token from the URL query string.

4. Backend

4.1 Application Bootstrap — main.py

FastAPI application configured with:

  • Lifespan handler that creates all DB tables on startup
  • CORS restricted to FRONTEND_ORIGIN
  • In-memory rate limiter on all /api/auth/* routes (20 requests / 60 s per IP)
  • Security headers middleware: X-Content-Type-Options, X-Frame-Options, Strict-Transport-Security, X-XSS-Protection
  • Two utility endpoints: GET /health and GET /api/procedures

4.2 Configuration — config.py

Pydantic Settings class, reads from environment / .env file.

VariableRequiredDescription
DATABASE_URLYesPostgreSQL connection string
JWT_SECRETYesHMAC signing key for JWTs
JWT_EXPIRY_DAYSNoCookie lifetime (default 30)
JWT_ALGORITHMNoDefault HS256
CLAUDE_API_KEYNoEnables AI analysis feature
FRONTEND_ORIGINYesCORS allowed origin(s)
NODE_ENVNodevelopment / production / test
SMTP_HOST / PORT / USER / PASSNoPassword-reset emails
Note: If DATABASE_URL starts with postgres:// it is automatically rewritten to postgresql+asyncpg:// for the async driver.

4.3 Database — database.py

Async SQLAlchemy engine with connection pooling. Provides:

  • engine — async engine instance
  • AsyncSessionLocal — session factory
  • Base — declarative base for ORM models
  • get_db() — FastAPI dependency that yields a session and commits / rolls back

5. API Reference

5.1 Auth — /api/auth

POST/api/auth/registerCreate account and auto-login
POST/api/auth/loginAuthenticate with username or email
POST/api/auth/forgot-passwordSend reset email (always returns 204)
POST/api/auth/reset-passwordSet new password via reset token
POST/api/auth/logoutClear auth cookie
GET/api/auth/meReturn current user profile
Auth endpoints set an httpOnly; Secure; SameSite=Lax cookie named access_token. Rate limiting: 20 requests / 60 s per IP.

5.2 Cases — /api/cases

GET/api/casesPaginated list with filters
GET/api/cases/{id}Single case detail
POST/api/casesCreate new case
PUT/api/cases/{id}Update existing case
DELETE/api/cases/{id}Delete single case
POST/api/cases/bulk-deleteDelete multiple cases by ID list

GET /api/cases — Query Parameters

ParameterTypeDescription

categorystringObstetrics | Gynaecology | ART
procedurestringFilter by procedure name
pregnantboolFilter by pregnancy outcome
date_fromdateStart of date range
date_todateEnd of date range

page_sizeintItems per page: 10 / 25 / 50 / 100
ART pregnancy logic: When a case is created or updated with category=ART and pregnant=true, pregnancy_check_date is automatically set to case_date + 14 days.

5.3 Users — /api/users

GET/api/users/profileGet current user
PUT/api/users/profileUpdate name, email, username, clinical_role
POST/api/users/change-passwordChange password (requires current password)
GET/api/users/qualificationsList user qualifications
POST/api/users/qualificationsAdd a qualification
DELETE/api/users/qualifications/{id}Remove a qualification
GET/api/users/hospitalsList hospital autocomplete options
POST/api/users/hospitalsAdd a hospital to autocomplete

5.4 AI — /api/ai

POST/api/ai/analyzeAnalyse cases with Claude Opus 4.6

Request Body

{
  "cases": [ ...CaseResponse[] ],   // max 500 cases
  "prompt_type": "reflective"       // summary | competency | reflective
}
Requires CLAUDE_API_KEY to be set in the environment. The key is never returned to the frontend. Timeout: 60 seconds.

5.5 Export — /api/export

GET/api/export/csvDownload all cases as CSV
GET/api/export/excelDownload all cases as styled XLSX

Both formats include: Case ID, Patient ID, Date, Hospital, Clinical Role, Category, Type, Procedure, Detail, Observations, Outcome, Pregnant, Complications, Previous CS, Sterilisation, Pregnancy Check Date, Oocyte Data, ET Data, Extra Data, Created At.

Excel extras: blue header row, alternating row shading, frozen header, auto-sized columns (max 60 chars).

5.6 Utility Endpoints

GET/healthHealth check — returns {"status": "ok"}
GET/api/proceduresFull procedure hierarchy (public, no auth required)

7. Security

ConcernImplementation
Authentication JWT stored in httpOnly; Secure; SameSite=Lax cookie. Not accessible to JavaScript.
Password storage SHA-256 pre-hash → bcrypt with auto-salt. Pre-hashing handles passwords > 72 bytes (bcrypt limit).
Rate limiting In-memory counter per IP. Auth routes capped at 20 req / 60 s.
CORS Restricted to FRONTEND_ORIGIN env var. Credentials mode enabled.
SQL injection SQLAlchemy ORM with parameterised queries throughout. No raw SQL strings.
User isolation Every case / qualification / hospital query includes WHERE user_id = :current_user_id.
Security headers Strict-Transport-Security, X-Content-Type-Options: nosniff, X-Frame-Options: DENY, X-XSS-Protection: 1; mode=block
Password reset HMAC token valid for 1 hour. Token is invalidated immediately on password change by incorporating the password hash into the signing material.
AI key exposure CLAUDE_API_KEY is server-side only. Never returned in any response or logged.
Forgot password Endpoint always returns 204 regardless of whether the email exists (prevents enumeration).

8. Deployment

Docker Compose Services

ServiceImage / BuildPortNotes

Production network

Both backend and frontend attach to an external docker-shared-net bridge network. This allows Nginx on the host to reverse-proxy to the containers.

Local Development

# Start all services (including local Postgres)
docker compose -f docker-compose.local.yml up

# Backend available at:  http://localhost:8000
# Frontend available at: http://localhost:18000

Production Deployment

# Build and start
docker compose up -d --build

# View logs
docker compose logs -f backend

# Database backup
docker exec <db-container> pg_dump -U postgres prolog > backup.sql

Environment Variables (.env)

DATABASE_URL=postgresql+asyncpg://user:pass@db:5432/prolog
JWT_SECRET=your-secret-key-here
JWT_EXPIRY_DAYS=30
CLAUDE_API_KEY=sk-ant-...          # optional
FRONTEND_ORIGIN=https://your-domain.com
SMTP_HOST=smtp.example.com         # optional
SMTP_PORT=587
SMTP_USER=noreply@example.com
SMTP_PASS=smtp-password
NODE_ENV=production
Never commit .env to version control. Use .env.example as a template and add .env to .gitignore.

Nginx Configuration (production)

Nginx terminates SSL (Let's Encrypt), proxies /api/* requests to the FastAPI container, and serves the React SPA static build for all other paths. The SPA fallback (try_files $uri /index.html) is required for client-side routing to work.
